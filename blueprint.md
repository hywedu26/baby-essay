# 육아 에세이 웹앱 개발 Blueprint

## 1. 프로젝트 개요

이 프로젝트는 아기 사진과 기록을 바탕으로 육아 에세이를 생성, 편집, 관리할 수 있는 웹 애플리케이션을 개발하는 것을 목표로 합니다. 사용자의 글쓰기 스타일을 학습하여 개인화된 결과물을 제공하며, 사용자와 상호작용하는 현대적이고 아름다운 UI를 제공합니다.

## 2. 핵심 기능

- **로그인 및 개인화:**
    - 아이디와 비밀번호를 이용한 사용자 인증.
    - 로그인 시, 등록된 아기 프로필 사진 또는 가족 사진으로 사용자 환영.
- **육아 기록 및 에세이 생성:**
    - 특정 날짜를 선택하여 아기 사진 업로드 및 간단한 기록 작성.
    - 업로드된 사진과 기록을 기반으로 AI가 육아 에세이 초안 생성.
    - 생성된 에세이를 사용자가 자유롭게 편집 및 저장.
- **AI 톤앤매너 학습:**
    - 초기 설정 시, 사용자가 기존에 작성한 육아 에세이 3편을 제출.
    - AI가 제출된 글의 톤앤매너(문체, 감성, 어휘 등)를 학습하여 이후 에세이 생성에 반영.
- **날짜 선택 기능:**
    - 기본 작성일은 현재 날짜로 설정.
    - 사용자가 원하는 과거 날짜를 선택하여 기록을 추가하거나 수정 가능.

## 3. 기술 스택 및 디자인

- **프론트엔드:** HTML, CSS, JavaScript (ES Modules, Web Components)
- **디자인:**
    - **Aesthetics:** 모던하고 따뜻한 느낌의 디자인을 적용하여 사용자에게 편안하고 즐거운 경험을 제공합니다. 부드러운 색상 팔레트, 감성적인 타이포그래피, 고품질 아이콘 및 이미지를 사용하여 시각적 매력을 극대화합니다.
    - **Layout:** 모바일과 웹에서 모두 완벽하게 작동하는 반응형 레이아웃을 구현합니다.
    - **Interactivity:** 버튼, 입력 필드, 슬라이더 등 모든 인터랙티브 요소에 그림자 효과와 미묘한 애니메이션을 추가하여 사용성과 시각적 만족도를 높입니다.
    - **Accessibility:** 모든 사용자가 쉽게 접근하고 사용할 수 있도록 웹 접근성(A11Y) 표준을 준수합니다.

## 4. 개발 계획

- **1단계: 기본 구조 및 로그인 화면 구현 (완료)**
    - `index.html`: 앱의 기본 HTML 구조 설정. 로그인 폼과 메인 콘텐츠 영역 분리.
    - `style.css`: 로그인 화면 및 전체적인 레이아웃 스타일링.
    - `main.js`: 기본적인 로그인 기능 구현.

- **2단계: 육아 기록 및 사진 업로드 기능 구현 (진행중)**
    - **`index.html` 수정:**
        - 날짜 선택 input (`type="date"`) 추가.
        - 텍스트 기록을 위한 `textarea` 추가.
        - 사진 업로드를 위한 `input` (`type="file"`) 및 미리보기 `img` 태그 추가.
        - 기록 저장 버튼 추가.
    - **`style.css` 수정:**
        - 새로 추가된 폼 요소들을 전체적인 디자인 톤에 맞게 스타일링.
        - 업로드된 사진 미리보기 스타일 적용.
    - **`main.js` 수정:**
        - 사진 업로드 시 미리보기 기능 구현.
        - ‘저장’ 버튼 클릭 시, 날짜, 기록, 사진 데이터를 객체로 묶어 화면 하단에 리스트 형태로 표시하는 기능 구현. (초기에는 로컬 메모리에 저장)

- **3단계 (예정):** AI 에세이 생성 기능 연동 (초기에는 Mock 데이터 사용).
- **4단계 (예정):** 톤앤매너 학습 기능 구현.
- **5단계 (예정):** 에세이 편집 및 저장 기능 구현.
